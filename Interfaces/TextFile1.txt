Интерфейсы
В ООП интерфейс - разновидность абстрактного класса, у которого:
	- все элементы public;
	- все абстрактные;
	- только методы (и свойства);

Другие значения этого термина: 
 - правило взаимодействия (подключения)		- USB-интерфейс
 - взаимодействие с пользователем			- UI (User Interface)
 - поставщики данных						- например, Web Interface
 - "видимая" public часть модуля			- (библиотеки или объекта, пространства имен) прототипы видимых функций

Зачем нужны интерфейсы (ООП)
Абстракция - синоним "обобщения", "собирательный термин", действие, обратное наследованию, расширению. 
Абстрагирование - создание обобщений, родительских классов (абстракций), объединяющих конкретные сущности.

Интерфейсы считаются высшей формой абстракции - объединяют сущености по минимальному кол-ву признаков в т.ч. и вообще без признаков - интерфейсы-маркеры
С "обратной стороны", появляется возможность разделить сущности на группы, которые реализуют или не реализуют данные интерфейсы
Причем одна сущность можеть быть участником нескольких групп, реализуя разные интерфейсы
